Below the Macintosh conventions will touch the fact that the colon ":" 
is an invalid character in the Macintosh Operating Systems. 
Under Windows and the NTFS file system the colon is an illegal character, 
because it is used to open alternate file streams. 

The only illegal character for file and folder names in Mac OS 9 is the colon ":"

File and folder names are not permitted to begin with a dot "."

File and folder names may be up to 255 characters in length
